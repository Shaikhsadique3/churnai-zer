üìã Configuration Options

Initialization Options

Churnaizer.init({
    // Required
    apiKey: 'your_api_key',
    
    // CSS selector(s) for cancel elements
    selector: '.cancel-btn, [data-cancel], .delete-account',
    
    // Auto-attach to dynamic elements
    autoTrack: true,
    
    // Debug mode
    debug: false,
    
    // Fallback function if modal fails
    fallback: function(event, element, error) {
        console.log('Cancel guard failed, proceeding with action');
        // Custom fallback logic
    }
});

Available Methods

  • Churnaizer.init(options) - Initialize with configuration
  • Churnaizer.setUser(userData) - Update user information
  • Churnaizer.setContext(context) - Update context data
  • Churnaizer.track(eventType, properties) - Track custom events
  • Churnaizer.trigger(userData, context) - Manually show modal
  • Churnaizer.getConfig() - Get current configuration
  • Churnaizer.isInitialized() - Check initialization status

üîß Advanced Features

Custom Fallback Handling

Churnaizer.init({
    apiKey: 'your_key',
    selector: '.cancel-btn',
    fallback: function(event, element, error) {
        // Log the error
        console.error('Cancel guard failed:', error);
        
        // Show custom message
        alert('Sorry, there was an issue. Please contact support.');
        
        // Or proceed with original action
        if (element.href) {
            window.location.href = element.href;
        }
    }
});

Dynamic User Data

// Update user data based on current state
Churnaizer.setUser({
    id: getCurrentUserId(),
    mrr: getCurrentMRR(),
    plan: getCurrentPlan(),
    tenure_days: getTenureDays(),
    last_login_days: getLastLoginDays()
});

// Track custom properties
Churnaizer.track('billing_page_viewed', {
    page: 'billing_settings',
    plan_type: 'pro',
    payment_method: 'stripe'
});
